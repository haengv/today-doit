## 1. 개요 (Overview)

### 서비스명
**Pudding** — 오늘을 버티게 하는 달달한 보상 TO DO

### 한 줄 소개
> 오늘 해야 할 일을 작은 보상과 연결해, 사용자의 행동 시작을 돕는 서비스

### 배경 및 문제 정의

사람들은 해야 할 일을 알고 있음에도 **즉각적인 보상이 없다는 이유로 행동을 미루는 경우**가 많다. 먼 미래의 성취(취업 성공, 자격증 합격 등)는 동기 부여로서 너무 추상적이며, 오늘 당장 행동을 시작하게 만드는 힘이 부족하다.

**Pudding**은 이 간극을 해소한다. 거창한 목표가 아닌, **하루를 버티게 하는 달달한 작은 행복(푸딩)**을 미리 설정하고, 할 일을 완료할 때마다 그 보상을 시각적으로 획득하는 경험을 제공한다.

---

## 2. 목표 (Goals)

### 제품 목표
- 사용자가 할 일을 등록할 때 **"완료 후 얻을 보상"을 함께 설정**하는 습관을 만든다.
- 할 일 완료 시 즉각적인 보상(푸딩 획득)으로 **도파민적 피드백**을 제공한다.
- 획득한 푸딩을 모아볼 수 있게 하여 **지속적인 동기 부여**를 유지한다.

### 비즈니스/검증 목표
- 먼 미래 성취가 아닌 **오늘의 작은 보상**과 연결했을 때 행동 시작률이 높아지는지 검증한다.
- MVP 단계에서 핵심 지표를 통해 서비스 가설을 확인한다.

## 3. 타겟 사용자 (Target Users)

### 주요 타겟
- **연령**: 10대 ~ 30대
- **특성**: 해야 할 일을 자주 미루는 사람
- **상황**: 공부, 취업 준비, 자기계발 중인 사람

### 사용자 페르소나

| 항목 | 내용 |
|---|---|
| **이름** | 김지은 (24세, 취업 준비생) |
| **상황** | 매일 자소서 쓰기, 영어 공부 등 해야 할 일은 명확하지만 시작이 어려움 |
| **고통** | "오늘도 못 했어" 반복되는 자책감, 동기 부여 고갈 |
| **욕구** | 작은 성취감, 오늘을 잘 버텼다는 느낌 |
| **보상 예시** | 할 일 끝내고 좋아하는 카페 가기, 드라마 한 편 보기, 마라탕 먹기 |

> **"해야 할 일을 먼 미래의 성취가 아닌 오늘의 작은 보상과 연결한다면, 행동 시작률이 높아지고 할 일 완료율이 개선될 것이다."**

## 5. 핵심 기능 (Core Features)

### F1. 오늘의 할 일 작성
- 사용자는 오늘 해야 할 일을 텍스트로 입력한다.
- 여러 개의 할 일을 추가할 수 있다.
- 각 할 일에는 보상(푸딩) 1개가 연결된다.

### F2. 오늘의 작은 보상(푸딩) 설정
- 할 일과 1:1로 대응되는 보상을 텍스트로 입력한다.
- **예시 키워드 선택 기능**: 카테고리별 예시 태그를 제공하여 빠르게 선택 가능하다. 직접 입력도 가능하다.

#### 보상 예시 키워드 카테고리

| 카테고리 | 예시 키워드 |
|---|---|
| ☕ **카페 / 음료** | 좋아하는 카페 가기, 버블티 사기, 아이스 아메리카노 한 잔 |
| 🍜 **음식** | 마라탕 먹기, 편의점 간식 털기, 배달 음식 시키기, 디저트 사 먹기 |
| 🎬 **콘텐츠** | 드라마 한 편 보기, 유튜브 실컷 보기, 영화 보기, 좋아하는 웹툰 정주행 |
| 🛁 **휴식** | 따뜻하게 목욕하기, 낮잠 자기, 아무것도 안 하기, 일찍 자기 |
| 🛍️ **쇼핑** | 갖고 싶던 거 하나 사기, 쇼핑 앱 구경하기, 스티커 사기 |
| 🎮 **게임 / 취미** | 게임 한 판 하기, 플레이리스트 만들기, 좋아하는 음악 듣기 |
| 🌿 **자기 관리** | 마스크팩 하기, 좋아하는 향초 켜기, 산책하기, 스트레칭 하기 |
| 💌 **소소한 기쁨** | 친구한테 연락하기, 좋아하는 사진 저장하기, 일기 쓰기 |

### F3. 시각화 — 빈 접시 → 푸딩이 채워지는 경험
- 할 일이 **미완료** 상태일 때: 빈 접시(푸딩이 없는 상태)로 표시
- 할 일이 **완료** 시: 빈 접시에 푸딩이 채워지는 애니메이션 제공
- 각 할 일 카드에는 다음 문구가 표시됨:
  > _"[할 일] 완료 시 → [보상]을 얻을 수 있어요 🍮"_

### F4. 할 일 완료 체크 및 푸딩 획득
- 사용자가 할 일을 완료하면 체크 버튼을 누른다.
- 완료 시 **푸딩 획득 애니메이션/피드백**이 발생한다.
- 완료된 할 일은 시각적으로 구분된다 (취소선, 색상 변경 등).

### F5. 획득한 푸딩 모아보기
- 사용자가 지금까지 획득한 푸딩 목록을 한눈에 확인할 수 있다.
- 날짜별로 획득한 보상과 완료한 할 일을 함께 표시한다.
- "오늘 N개의 푸딩을 획득했어요!" 같은 요약 메시지 제공.

## 6. 사용자 플로우 (User Flow)

```
[앱 진입]
    ↓
[오늘의 할 일 입력]
    → 할 일 텍스트 작성
    → 연결할 보상(푸딩) 설정 (예시 키워드 or 직접 입력)
    ↓
[오늘의 할 일 목록 화면]
    → 각 할 일 카드: 빈 접시 + "완료 시 [보상] 획득" 문구
    → 할 일 완료 → 체크 버튼 클릭
    ↓
[푸딩 획득 애니메이션]
    → 접시에 푸딩이 채워짐
    → "🍮 [보상]을 획득했어요!" 메시지
    ↓
[푸딩 모아보기]
    → 오늘 / 전체 획득 푸딩 목록 확인
```

## 9. 디자인 방향 (Design Direction)

### 비주얼 컨셉
- **키워드**: 달달함, 따뜻함, 귀여움, 작은 행복
- **색상**: 크림/오프화이트 베이스, 노랑(#FFD84D)·주황(#F4A23A) 포인트 컬러
- **일러스트 스타일**: 손으로 그린 듯한 스케치 선(두꺼운 흑색 아웃라인), 컬러는 캐러멜 소스·스파클 포인트에만 제한적으로 사용
- **대표 캐릭터**: 아래 확정된 푸딩 일러스트 기반

![pudding character](/Users/haeng/.gemini/antigravity/brain/e80c930c-aa5f-41ea-b157-19ed4360a19c/pudding_character_1780926102758.png)

### UX 원칙
- **진입 장벽 최소화**: 할 일 입력 → 보상 설정이 3 tap 이내로 완료
- **즉각적 피드백**: 완료 체크 시 즉각적인 시각적/감각적 보상 피드백
- **긍정 강화**: 자책이 아닌 달성에 집중하는 언어 사용 ("잘 했어요!", "오늘의 푸딩을 얻었어요!")

## 10. 범위 외 사항 — MVP (Out of Scope)

MVP 단계에서는 아래 항목을 포함하지 **않는다**.

- 소셜 기능 (친구와 공유, 랭킹 등)
- 알림/리마인더 기능 (푸시 알림)
- 반복 할 일 (매일/매주 반복)
- 푸딩 커스터마이징 (색상, 모양 선택 등)
- 로그인/회원가입 ✅ **로컬 스토리지로 운영 확정**

## 13. 데이터 구조 (Local Storage Schema)

```ts
// 할 일 (Todo)
interface Todo {
  id: string;           // uuid
  date: string;         // "YYYY-MM-DD"
  task: string;         // 할 일 내용
  reward: string;       // 보상(푸딩) 내용
  completed: boolean;   // 완료 여부
  completedAt?: string; // 완료 시각 (ISO 8601)
  createdAt: string;    // 생성 시각 (ISO 8601)
}

// 로컬 스토리지 키
// pudding_todos: Todo[]
```

> [!NOTE]
> 자정 초기화는 앱 진입 시 오늘 날짜와 저장된 날짜를 비교하여 처리한다. 이전 날짜 데이터는 삭제하지 않고 아카이브로 유지한다.
